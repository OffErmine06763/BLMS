plot_bar_perc <- function(vars, scale = NULL) {
  x_scale <- if (!is.null(scale)) {

    NULL
  }

      fill = Satisfaction

    geom_text(
      stat = "count",

    ) +

}

Initial Regression Model

Start by fitting a regression model using only the predictors

\[\texttt{Age},\ \texttt{SeatComfort},\ \texttt{FlightDistance},\ \texttt{Class},\ \texttt{DepartureDelay},\ \texttt{ArrivalDelay}\]

First of all, BAS is using BMA (Bayesian Model Averaging), so we train many models and give them a weight.

To interpret the summary:

  • P(B != 0 | Y): aka Posterior Inclusion Probability (PIP), how likely the variable belongs to the true model

  • The 1/0 under the models for the first rows (those corresponding to the predictors): boolean flag, whether that predictor is included in the model

  • BF: Bayes Factor, idk what it is but it’s a metric that compares each model with the best one, the higher the better, the best model has BF = 1 by construction.

  • PostProb: probability that that model is the true one, given the data

  • logmar: the higher the better

From the summary we see that:

  • Flight.Distance and Departure.Delay are not selected very frequently across all the models trained by BAS, they might not capture useful data to fit the Satisfaction

  • The best model obtained is much better than the others: PostProbs and BF are much higher than the other models, meaning that the data strongly prefers model 1 over the other alternatives.

  • R2 is very bad for all the models, we are not fitting the data at all

In the summary of coefficients reports the average over the evaluated models weighted by their posterior probability, where

  • post mean: average effect of a variable across all models, but models where the variable is absent count as 0 effect, so this is a shrunken, uncertainty-aware estimate.

  • post SD: uncertainty on the effect, including the model uncertainty

so we can say that

  • Age has a positive effect and high PIP: older passengers have higher probability of satisfaction; the same goes for Seat.Comfort, as expected

  • Arrival.Delay has a negative effect (as expected) and discrete PIP, so there is some uncertainty in whether this predictor is actually important.

  • Departure.Delay has almost 0 mean and very poor PIP, this is probably due to the fact that once Arrival.Delay is included, Departure.Delay adds little information.

  • Flight.Distance: same as for Departure.Delay, this is probably irrelevant because people know the flight length in advance.

Seat.Comfort as Categorical

Much slower but gives a better R2, however there are some weird things. We can expect a model deciding to exclude Seat.Comfort as a whole to make predictions, however here (since it’s treated as 5 distinct variables) it can happen (as in model 1) that only a subset of the levels is selected, and others are ignored. Now, if the selected ones where all the k highest levels it would be fine (it means “if the score is above below a certain value, it might as well be treated as a 0 (which is the baseline level)”), however we see that level 5 is ignored, which means “if the observation has score = 5, it is equivalent to score = 0” which is clearly wrong. The solutions to this are three: group priors (idk what they are), hierarchical structure (idk what it is) or treating the factors as numeric, as done above.

This still has a better R2 (not that it was hard to be better than the previous model), but i still struggle to understand the residual plot.
